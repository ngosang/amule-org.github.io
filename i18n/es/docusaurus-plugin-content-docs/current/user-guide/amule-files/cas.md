CAS (Client Animation Script) is a utility shipped with aMule that generates a status **image** and/or **HTML page** reflecting aMule's current state. It reads data from [`amulesig.dat`](./amulesig-dat.md) and uses user-supplied templates and font settings to produce its output.

The files in this page are created only when CAS has been run.

## `casrc`

**Location:** `~/.aMule/casrc`

Configuration file for CAS. It controls which font to use, which background image to draw text onto, the path to an HTML template, and where each line of status text should be placed in the image.

This file is created the first time CAS is run with the `-o` or `-p` switch. It can be edited manually.

### Format

Plain text. Each non-empty, non-comment line is either a **parameter** or a **line placement definition**.

Lines beginning with `#` are comments and are ignored.

#### Parameters

| Parameter | Value | Description |
|---|---|---|
| `font` | Path to a `.ttf` file | Font used to draw text in the generated image |
| `font_size` | Floating-point number | Size of the font (e.g., `10.5`) |
| `source_image` | Path to an image file | Background image on which the text is rendered |
| `template` | Path to an `.html` file | HTML template used to generate the HTML status page |
| `img_type` | `0` or other integer | Output image format: `0` = PNG, any other value = JPG |

#### Line placement definitions

After the parameter lines, CAS reads lines that define where each piece of status text is placed in the image. The first seven placement lines encountered are used (one per status line displayed); any additional placement lines are ignored.

Format of a placement line:

```
<identifier>  <x>,<y>,<enable>
```

| Field | Description |
|---|---|
| `identifier` | Any text label (CAS ignores it; it is for human readability only) |
| `x` | Pixels from the **left** edge of the image to the start of the text |
| `y` | Pixels from the **top** edge of the image to the baseline of the text |
| `enable` | `1` to draw this line; `0` to skip it |

#### Default `casrc`

```
# cas config file
#
# font         - full path to a ttf font
# font_size    - size the font
# source_image - image where the text will be written
# *_line       - x,y,[1/0] enabled or disabled

font        /usr/share/fonts/corefonts/times.ttf
font_size   10.5
source_image /usr/share/cas/stat.png

first_line   23,17,1
second_line  23,34,1
third_line   23,51,1
fourth_line  23,68,1
fifth_line   23,85,1
sixth_line   23,102,1
seventh_line 23,119,1

template /usr/share/cas/tmp.html
img_type 0
```

Default paths used by CAS on a typical Linux installation:

| Resource | Default path |
|---|---|
| Font | `/usr/share/fonts/corefonts/times.ttf` |
| Background image | `/usr/share/cas/stat.png` |
| HTML template | `/usr/share/cas/tmp.html` |

## `aMule-online-sign.html`

**Location:** `~/.aMule/aMule-online-sign.html`

An HTML page generated by CAS that shows aMule's current status. This file exists only if CAS has been asked at some point to generate an HTML output page.

The page is produced from the HTML template referenced by the `template` parameter in `casrc`, with status values filled in from [`amulesig.dat`](./amulesig-dat.md).

## `aMule-online-sign.png`

**Location:** `~/.aMule/aMule-online-sign.png`

A PNG image generated by CAS showing aMule's current status as text drawn onto the background image defined in `casrc`. This file exists only if CAS has been run at some point with the `-o` switch **and** `casrc` was successfully configured.

The image is produced by:
1. Loading the background image from `source_image`
2. Drawing up to seven lines of status text at the positions defined by the placement lines in `casrc`, using the configured font and size
3. Saving the result as PNG or JPG (controlled by `img_type` in `casrc`) to `~/.aMule/aMule-online-sign.png`

The status lines are taken directly from the corresponding lines of [`amulesig.dat`](./amulesig-dat.md).
