aMule 3.0.0 is out — the first major release in over five years (since 2.3.3 in February 2021). This release brings dramatic throughput improvements, a full build-system overhaul, a modernized dependency stack, native binaries for Linux, macOS, and Windows, and a broad legacy-API cleanup.

<!-- truncate -->

## Highlights

- **Throughput rewrite.** Disk I/O moved off the main thread, ASIO/EPOLLET races fixed, throttlers replaced with proper token-bucket limiters. Peer-to-peer download on the same hardware sees **~100–380× speedups** across macOS / Linux / Windows over 2.3.3, plus aMule 3.0.0 sustains **~4.8×** the upload throughput of eMule 0.70b on Windows.
- **CMake replaces autotools.** Single build system, modern toolchain — minimum CMake 3.10, minimum wxWidgets 3.2.0.
- **Native binaries for every major desktop.** AppImage (x86_64 + aarch64), Flatpak (x86_64 + aarch64), macOS Universal2 .dmg, Windows portable .zip (x64 + ARM64).
- **HTTPS works again.** `CHTTPDownloadThread` rewritten on top of `wxWebRequest`; the hand-rolled stack had silently stopped working against modern TLS.
- **Kad parallel searches** with alpha-frontier widening.
- **MaxMindDB replaces deprecated GeoIP** for IP→country.

## Performance

Throughput is the headline story of this release.

### aMule 3.0.0 vs 2.3.3

| Leecher platform | 2.3.3 | 3.0.0 | Speedup |
|---|---:|---:|---:|
| macOS (Apple Silicon, Mac Studio) | 0.35 MB/s | **135 MB/s** | **381×** |
| Linux ARM (UTM VM, Ubuntu 25.10) | 0.34 MB/s | **117 MB/s** | **345×** |
| Windows ARM (UTM VM, Windows 11) | 0.36 MB/s | **39 MB/s** | **107×** |

Measured over a 90 s window, single LAN peer downloading a 30 GB file from an x86_64 Linux seeder.

### aMule 3.0.0 vs eMule 0.70b (Windows)

| Direction | eMule 0.70b | aMule 3.0.0 | Speedup |
|---|---:|---:|---:|
| **Upload** (Windows seeds → Mac leecher) | 22 MB/s | **106 MB/s** | **~4.8×** |
| **Download** (Linux seeds → Windows leeches) | 20 MB/s | **39 MB/s** | **~1.9×** |

The gains come from several complementary fixes:

- **`CUploadDiskIOThread` port** (#451) — disk reads and ed2k packet construction moved off the main thread; RC4 stream-desync race and EPOLLET spurious-wakeup fixed. Dominant contributor to the upload gain.
- **`MaxUpload=0` is now literal unlimited** (#461) — the old code tracked `current_rate + 5 KB/s` per tick, capping the uplink far below capacity.
- **`CPartFileWriteThread`** (#454) — disk writes for downloads moved off the main thread.
- **Throttler-wake fix** (#484) — the upload throttler adaptive backoff dozed 5–25 ms between control-queue pumps; an empty→non-empty wake gate eliminates the stall.
- **`MaxDownload` is now a literal cap** (#491) — replaced the old ratio controller with a global token bucket enforcing the configured limit to within 2.5%.
- **Speed-limit fields widened from `uint16` to `uint32`** (#436) — the old 65,534 KB/s (~524 Mbps) ceiling is gone; gigabit-class links can now be configured correctly.

## Networking & Discovery

- **Kad parallel searches** with alpha-frontier widening; new "More" button to fetch results from additional peers (#505).
- **MaxMindDB replaces deprecated GeoIP** for IP→country (`.dat` → `.mmdb`; free MaxMind account required for the database) (#502, #507, #509).
- HTTPS downloads work again — `CHTTPDownloadThread` rewritten on top of `wxWebRequest` (#462).
- Wire-parser hardening: `OP_SERVERMESSAGE` size validation (#447), buffer overflow fix in `ECSocket` (#421), uninitialized memory fix in `UInt128` (#424).

## Build System & Dependencies

aMule 3.0 requires **CMake ≥ 3.10** and **wxWidgets ≥ 3.2.0**. Autotools is removed entirely.

- Full CMake build replaces autotools (#449). Autotools and the `debian/` packaging dir retired (#466).
- wxWidgets 3.x build fixes (#438, #443, #453); macOS framework / app-bundle compat (#453).
- Boost 1.87 / 1.89 build fixes (#387, #429).
- CMake hard-fails when `ENABLE_X=YES` cannot be honored — no more silent disables.
- Loongarch64 architecture support (#286).

## Packaging

aMule 3.0 ships native binaries covering all modern desktops:

- **Linux AppImage** (x86_64, aarch64) — works across all major distros; first-run prompt installs a desktop launcher.
- **Linux Flatpak** (x86_64, aarch64) — runs in the GNOME Platform sandbox.
- **macOS Universal2 .dmg** — Apple Silicon and Intel in a single bundle.
- **Windows portable .zip** (x64, ARM64).

Other platform integration: Wayland-friendly desktop integration with matching `app_id`, SNI tray icon (replaces deprecated `GtkStatusIcon`), macOS dock-restore on quit (#508).

## Bug Fixes & Stability

- `CFile::doSeek` no longer asserts when `IsOpened()` is false (#294).
- Selected lines in Download/Shared lists no longer become illegible when losing focus (#385).
- amulegui: clean startup on Cancel + watchdog on wrong connection data (#465).
- amuleweb: missing `prefs apply` options restored (#419).
- `PartFileWriteThread` catches `CIOFailureException` so disk-full no longer kills the process (#499).

## Known Limitations

- macOS `.dmg` is **not yet code-signed or notarized** — Gatekeeper warnings will appear on first launch.
- **Flathub submission pending** — install the `.flatpak` directly with `flatpak install <file>` for now.
